# WC-Bet-Advisor 开发需求文档(PRD)

> 版本: v1.0 · 日期: 2026-06-10 · 状态: 草案

## 1. 项目概述

### 1.1 项目名称
**WC-Bet-Advisor** — 2026 世界杯体彩竞猜购彩建议程序(Web 端)

### 1.2 业务背景
2026 年 FIFA 男足世界杯由美加墨三国联合举办,首次扩军至 **48 队 / 64+ 场**,赛期 **2026-06-11 至 2026-07-19**。中国体彩同步开售:

- **竞彩足球**:胜平负、让球胜平负、比分、总进球、半全场、混合过关
- **传统足彩**:胜负彩 14 场、任选九、6 场半全场、4 场进球彩

体彩规则公开但**不开放实时数据 API**,赔率由各博彩公司独立发布;海外预测市场 **Polymarket** 提供"无抽水"的市场隐含概率,可作为高质量交叉验证信号。

### 1.3 产品目标
为彩民提供一个**基于多源数据 + 价值识别 + 凯利公式**的购彩决策辅助工具,产出**有数学依据**的单注、串关、14 场胜负彩、任选九方案;并通过**截止时间雷达**和**跟单复盘**形成"决策 → 执行 → 复盘"闭环。

### 1.4 核心价值主张

| 维度 | 做法 |
|---|---|
| **价值识别** | ELO + 赔率 + Polymarket 三源融合,edge ≥ 5% 才进推荐池 |
| **风险控制** | 默认 1/4 凯利,单注封顶本金 5%,长周期博弈 |
| **截止提醒** | 顶部全局倒计时 + 桌面通知,绝不漏单 |
| **长期主义** | 跟单记账 + ROI 复盘,不看单场红黑 |

### 1.5 不做的事(V1 明确排除)

- ❌ 自动下注 / 真实资金对接
- ❌ 多用户系统 / 移动端
- ❌ 抓中国体彩官方实时盘口(合规)
- ❌ SP 值预测(只能赛后获取)
- ❌ 票面 OCR / 微信推送(放 V1.1)

## 4. 用户故事

> **US-01** 作为一个老彩民,我希望**今天有比赛时,打开网页就能看到剩余时间倒计时**,这样我不会错过购彩。
>
> **US-02** 作为一个注重数据的人,我希望**单场比赛能看到 ELO 差、市场概率、Polymarket 概率三组数**,这样我能判断推荐是不是有依据。
>
> **US-03** 作为一个买 14 场的用户,我希望**程序能直接给我一注 14 场方案 + 几注复式备选**,而不是让我自己拼 3^14 种组合。
>
> **US-04** 作为一个长期玩家,我希望**跟单记录能自动统计 ROI 和命中率**,这样我年底能算清自己是赚是赔。
>
> **US-05** 作为一个风险厌恶用户,我希望**凯利系数能调到 0.1 甚至 0**,这样我能保守地跟单。
>
> **US-06** 作为一个管理员(我),我希望**能批量手工导入赔率和截止时间**,因为体彩盘口变化时官方不通知。

## 5. 业务流程

### 5.1 购彩决策主流程

`
[打开网页]
   ↓
[顶部 banner: 距本期截止 02:14:33]
   ↓
[Dashboard 今日待购卡片流] → 选一场感兴趣
   ↓
[MatchDetail: 赔率对比 / 价值分析 / 凯利试算]
   ↓
[保存为方案] → [可选] 跟单记账
   ↓
[赛后] → [结算: 录入结果] → [PnL 自动更新]
`

### 5.2 数据更新流程

`
[APScheduler 每 6h]
   ↓
[拉 football-data.org 赛程/对阵]
[拉 Polymarket 市场(每 15min)]
[拉 Odds API 赔率]
[手工导入赔率(管理员触发)]
   ↓
[归一化 / 落库]
   ↓
[触发推荐重算]
   ↓
[推送通知: 新推荐 / 即将截止]
`

### 5.3 推荐生成流程

`
[开赛前 24h / 6h / 1h]
   ↓
[取该场所有赔率 + Polymarket + ELO]
   ↓
[三源融合 → p_final]
   ↓
[对每个玩法 × 每个选项算 edge / 凯利]
   ↓
[edge ≥ 5% 进推荐池]
   ↓
[按价值评分排序 → Top N]
   ↓
[串关: 贪心 + 相关性惩罚]
[14 场: 启发式搜索]
   ↓
[写入 recommendations 表 + 推前端]
`

## 8. 风险与约束

| 风险 | 影响 | 缓解 |
|---|---|---|
| **合规风险** | 中国体彩未授权实时数据 | 不抓官方,只读国际盘 + 手工 |
| **数据延迟** | 赔率非秒级,可能错过盘口变化 | 接受 6h 延迟,在 UI 标注数据时间戳 |
| **冷门预测** | 模型和市场一样会错 | 不假装能预测冷门,只推 edge 高的 |
| **截止时间误判** | 体彩中心可能临时调整 | 后台覆盖入口 + 偏差 > 1h 弹提醒 |
| **Polymarket 访问** | 国内可能受限 | 只读公开 API,Web 端用户不直连 |
| **预测 ROI 不正** | 历史回测可能为负 | 设置里公开回测结果,用户自决 |

## 9. 验收标准

### 9.1 功能验收

- [ ] F1-F10 全部 P0 功能可用,P1 功能至少 50% 完成
- [ ] 所有玩法(竞彩 6 + 传统 4)有推荐产出路径
- [ ] 截止时间倒计时准确,与体彩官网公告偏差 < 5 分钟
- [ ] 跟单记账数据准确,ROI 复盘结果与人工核算一致

### 9.2 数据验收

- [ ] 64 场赛程完整,小组 + 淘汰赛 + 决赛
- [ ] 每场比赛至少 3 家博彩赔率(手工导入)
- [ ] 热门场次 Polymarket 价格可拉取
- [ ] 历史回测(2018/2022)在 mock 模型上能跑通

### 9.3 质量验收

- [ ] 单元测试覆盖率 ≥ 60%
- [ ] 关键算法(凯利/泊松/价值)有专项测试
- [ ] Docker compose up 5 分钟内能跑通
- [ ] 任意页面 5xx 错误率 < 0.1%
